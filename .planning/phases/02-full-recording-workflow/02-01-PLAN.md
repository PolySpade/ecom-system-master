# Phase 02 Plan 01: Full Recording Workflow

**Goal:** Bring the recording loop to ecom-py parity: non-blocking save with a
"Saving Recording..." dialog and scan queueing (REC-04), Video Label routing
into folder/DB/watermark (REC-06/REC-07/STO-02/DB-03), burned-in FFmpeg
drawtext watermarks as a post-save step (CAM-05 via REC-06), camera
auto-reinit with cooldown (CAM-04), disk-space guard before recording
(STO-02/CAM-05), exit confirmation that stops-and-saves (UI-02), and the real
main window (UI-01/UI-03/UI-04): status panel with MM:SS ticker + storage
used + compression placeholder, recent-recordings list with Open File /
Show in Folder, 5s auto-clearing status bar, purple-seeded Material theme.

## Tasks

1. **FFmpeg locator + watermark pipeline** — `lib/core/ffmpeg_locator.dart`
   with the exact contract `FfmpegLocator.findFfmpeg()/findFfprobe()`
   (bundled `ffmpeg/bin/` beside exe -> common install paths -> PATH), and
   `lib/core/watermark_service.dart`: serialized post-save queue that
   re-encodes the saved file with three drawtext overlays (running timestamp
   top-left via `%{pts:localtime:EPOCH}`, label top-right on #667eea box,
   `Barcode: X` bottom-left), writes to a temp file and swaps with a backup
   so the original can never be lost; skips gracefully (logged warning) when
   FFmpeg or a usable font is absent. Pure filter/args builders unit-tested.

2. **Camera auto-reinit + richer stop result** — consecutive-failure counter,
   5s cooldown, non-reentrant reinit (ports `_try_reinitialize_camera`),
   periodic health monitor, salvage of an in-progress recording on failure;
   `StopRecordingResult` gains `videoPath`/`barcode`/`startTime` for the
   watermark job; plugin-output move uses rename-first (fast) with
   copy+delete fallback.

3. **Disk-space guard** — `lib/core/disk_space.dart`: Win32
   `GetDiskFreeSpaceEx` free-GB probe + pure `evaluateDiskSpace()`
   (critical=block below `storage.min_free_space_gb`, low=warn below 5x);
   new config key `storage.min_free_space_gb` (default 1.0; ecom-py has no
   free-space guard to copy a value from — documented decision).

4. **Main window + exit confirmation** — label dropdown (3 labels, default
   "Normal (Standard)"), scan queue (`lib/core/scan_queue.dart`, serialized
   drain loop so scans during a save are queued not dropped), non-dismissible
   saving dialog around stop+DB completion, DB stores basename (ecom-py
   parity) with `resolveVideoPath()` labeled->legacy fallback, status panel,
   recent list with Open File / `explorer.exe /select,` actions, status bar
   auto-clear 5s, seed #667EEA theme, `setPreventClose` + `onWindowClose`
   confirm-stop-save-exit.

5. **Tests + docs** — unit tests for filter/args construction, swap/skip
   behavior, disk guard, scan queue, path resolution; `flutter analyze`
   clean; summary doc.

## Out of scope

Compression pipeline (Phase 3 — placeholder status line only), search and
settings dialogs (Phase 4), hardware validation (needs camera + scanner).
